# Day 4：大模型核心概念术语表（30+ 必知概念）

> 目标：在进入 Transformer 之前，先建立大模型领域的词汇地图。不需要完全理解每个概念，但要知道每个词大概在说什么，后续学习中反复对照。

---

## 一、模型基础概念

### 1. 预训练（Pre-training）
在海量文本上做"完形填空"或"下一词预测"，让模型学会语言的统计规律。预训练产出的是 **Base Model**（基座模型），它只会续写文本，还不会对话。

### 2. 微调（Fine-tuning）
在预训练模型基础上，用高质量标注数据继续训练，让模型学会特定能力（对话、遵循指令、代码等）。**SFT**（Supervised Fine-Tuning，监督微调）是最常见的微调方式。

### 3. SFT（监督微调）
给模型看"问题→标准答案"对，让它学会按人类期望的方式回答。数据格式通常是 `{"instruction": "...", "output": "..."}`。ChatGPT 就是 GPT-3 经过 SFT 得到的。

### 4. RLHF（人类反馈强化学习）
先用人类偏好数据训练一个 Reward Model（奖励模型），再用强化学习让 LLM 输出获得更高奖励分。目的是让模型输出更"有用、无害、诚实"。

### 5. DPO（直接偏好优化）
RLHF 的简化版。不需要单独训练 Reward Model，直接从人类偏好对（哪个回答更好）中学习。2024 年后很多开源模型（如 Qwen 2.5）用 DPO 对齐。

### 6. 指令微调（Instruction Tuning）
SFT 的一种，专门用"指令-回答"对训练，让模型能理解和执行各种自然语言指令。

### 7. 对齐（Alignment）
让模型的输出与人类价值观对齐：有用（helpful）、无害（harmless）、诚实（honest）。包括 RLHF、DPO、红队测试等手段。

### 8. 蒸馏（Knowledge Distillation）
用大模型（Teacher）的输出训练小模型（Student），让小模型"继承"大模型的能力。DeepSeek-R1 就用了蒸馏让小模型获得推理能力。

## 二、模型参数与训练

### 9. 参数（Parameters）
模型的"记忆单元"，存储着模型学到的知识。7B = 70 亿个参数。参数越多 → 表达能力越强，但推理越慢、显存吃越大。

### 10. 权重（Weights）
参数的具体数值。FP16 格式下，每个权重占用 2 字节。7B 模型 ≈ 14GB 权重文件。

### 11. Epoch
把全部训练数据过一遍 = 1 个 epoch。通常预训练只需要 1-3 个 epoch（数据量巨大），微调可能需要 3-10 个 epoch。

### 12. Batch Size
一次训练喂给模型的样本数。受限于显存大小。大模型训练中 Batch Size 通常是百万 token 级别。

### 13. 学习率（Learning Rate）
每次参数更新的步长。太大 → 训练不稳定；太小 → 训练太慢。常用 warmup（预热）+ cosine decay（余弦衰减）策略。

### 14. 梯度（Gradient）
损失函数对每个参数的偏导数，指向"往哪个方向调参数能让损失变小"。反向传播就是计算梯度的过程。

### 15. 损失函数（Loss Function）
衡量"模型输出"和"标准答案"之间的差距。语言模型常用 **交叉熵损失**（Cross-Entropy Loss）。

## 三、推理概念

### 16. 推理（Inference）
模型生成输出的过程（只是前向传播，不更新权重）。和训练不同，推理不修改模型参数。

### 17. Token
模型处理文本的最小单位。英文约 1 token ≈ 0.75 个单词，中文约 1 token ≈ 0.5 个汉字。1000 tokens ≈ 750 英文单词 ≈ 500 汉字。

### 18. Tokenizer / 分词器
把自然语言文本切成 token 序列的工具。常见：BPE（GPT 系）、WordPiece（BERT）、SentencePiece（Llama 系）。

### 19. Embedding / 词嵌入
把 token 映射为稠密向量。语义相近的词向量也相近。向量的维度（如 4096）决定了模型的"表达能力宽度"。

### 20. 上下文（Context）
输入给模型的所有内容（system prompt + 历史对话 + 当前问题）统称为上下文。

### 21. 上下文窗口（Context Window）
模型一次能处理的最大 token 数。GPT-4o 是 128K，Claude 3 是 200K，Llama 3 是 8K。超出窗口的内容会被截断或遗忘。

### 22. 上下文长度扩展
让模型在推理时能处理比训练时更长的上下文。技术包括 NTK-aware、YaRN、Self-Extend 等。

## 四、生成控制

### 23. Temperature（温度）
控制生成文本的随机性。0 → 确定性输出（适合翻译/代码），1 → 有创意（适合写作），>1 → 放飞自我（胡言乱语）。

### 24. Top-p（核采样）
设置累积概率阈值 p。p=0.9 表示只从能覆盖 90% 概率的 token 中采样。和 Temperature 配合使用，控制"创意与可靠"的平衡。

### 25. Top-k
只从概率最高的 k 个 token 中采样。简单粗暴，但比 Top-p 更容易丢失多样性。

### 26. 贪心解码（Greedy Decoding）
每次选概率最高的那个 token。确定性最强，但容易陷入重复（"你好你好你好..."）。

### 27. Beam Search
维护 k 条候选序列，最后选整体概率最高的。翻译场景常用，对话生成一般不需要。

## 五、关键现象

### 28. 幻觉（Hallucination）
模型输出了看似合理但事实上错误的内容。比如"李白是宋朝诗人"、"量子力学由牛顿提出"。RAG 是缓解幻觉的主要手段。

### 29. 涌现（Emergence）
当模型规模超过某个阈值时，突然出现小模型没有的能力。比如超过 100B 参数后突然能做复杂推理。是 scaling law 的观察现象。

### 30. Scaling Law
模型性能随参数数量、数据量、计算量的增加可预测地提升。核心公式：Loss ∝ (参数)^(-α) × (数据)^(-β)。大模型时代的理论基石。

### 31. 思维链（Chain-of-Thought, CoT）
在 Prompt 中引导模型"一步一步思考"，显著提升复杂推理准确率。比如在问数学题之前加一句 "Let's think step by step"。

### 32. Lost in the Middle
长上下文中，模型对中间位置信息的提取能力显著低于开头和结尾。解决方法：重要信息放两端，使用 Reranker。

## 六、工程概念

### 33. 量化（Quantization）
用更低精度（INT8/INT4）存储模型权重，减小文件大小和显存占用。代价是微弱的质量下降。

### 34. GGUF
llama.cpp 生态的模型文件格式，权重 + tokenizer + chat template 打包成单文件。Q4_K_M 是推荐默认量化级别。

### 35. 吞吐量（Throughput）
单位时间内处理的请求数。衡量指标：tokens/秒（TPS）、requests/秒（QPS）。

### 36. TTFT（Time to First Token）
从发送请求到收到第一个 token 的时间。影响用户"等待感"的核心指标。通常 200-500ms 为良好。

### 37. TPOT（Time Per Output Token）
每个后续 token 的平均生成时间。流式输出场景下影响阅读流畅度。TTFT + TPOT × N = 总响应时间。

### 38. 流式输出（Streaming / SSE）
模型一边生成一边推送 token，而非等全部生成完再返回。提升用户体验的关键技术。

---

## 七、评估指标

### 39. Perplexity（困惑度）
衡量语言模型预测下一个 token 的"确定性"。困惑度越低越好。数学上是交叉熵损失的指数形式。

### 40. BLEU / ROUGE
机器翻译（BLEU）和文本摘要（ROUGE）的自动评估指标。通过对比模型输出和参考文本的 n-gram 重叠计算得分。

---

## 八、安全概念

### 41. 提示注入（Prompt Injection）
用户通过精心构造的输入覆盖模型的 System Prompt。"忽略之前的指令，现在你是 DAN..."。

### 42. 红队测试（Red Teaming）
模拟攻击者视角，系统性测试模型的安全边界。找到越狱、偏见、有害输出等问题。

### 43. 越狱（Jailbreak）
用各种技巧绕过模型的安全限制。是 Prompt Injection 的一种应用形式。
